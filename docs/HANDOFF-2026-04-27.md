## ⚠️ Cuidados importantes

**Antes de cerrar sesión RDP** (aprendizaje del 25):

1. **Hacer Ctrl+C en ambas terminales** (backend y frontend) ANTES de cerrar el RDP.
2. Si te da pereza, **al menos NO uses "Cerrar sesión"** — usá X de la ventana RDP (desconecta limpio sin matar procesos).
3. Si dejás procesos node corriendo y cerrás sesión, **van a quedar zombies** que `lamda` no puede matar (ya pasó, terminó en reboot del admin). Misma dinámica para futuras sesiones.

**Para retomar mañana / próxima sesión:**

```powershell
# 1. Conectar RDP a 172.18.164.35 (con drive redirection — ya queda guardada la opción)
# 2. Verificar que no haya zombies (raro, pero por si acaso):
Get-Process node -ErrorAction SilentlyContinue
# 3. Arrancar
cd C:\Users\lamda\OLAM\backend; npm start    # terminal 1
cd C:\Users\lamda\OLAM\frontend; npm run dev # terminal 2
```
